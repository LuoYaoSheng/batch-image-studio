请为一个 **桌面端图片批量局部处理工具** 生成一套 **完整、高保真、可直接指导后续实现** 的 UI 设计稿。

这是一个未来用于 **Tauri 跨平台桌面软件（Windows / macOS / Linux）** 的产品，**不是移动端，不是网页营销站，不是通用影像后期平台**。

请严格按下面要求输出，**不要自行扩展产品边界**。

## 一、最重要的产品定义

### 产品名称

**图片批量处理大师**

### 产品定位

这是一款 **用户自定义模板型的桌面端图片批量局部处理工具**。

用户在一张样图上：

1. 手动框选一个局部区域
2. 选择处理方式
3. 预览效果
4. 保存为模板
5. 再把模板批量应用到相似图片上

### 核心原则

只有两条：

1. **第一次使用尽量简单**
2. **模板建好后后续使用更简单**

### 主链路

#### 首次使用主链路

**导入图片 → 选样图 → 框选区域 → 选处理方式 → 预览效果 → 保存模板 → 批量处理**

#### 后续使用主链路

**导入图片 → 选模板 → 预览一张 → 批量处理**

### 页面设计原则

1. **每一页只做当前阶段最重要的事**
2. **不要让用户自己思考流程**
3. **默认简单，高级折叠**
4. **模板是唯一核心复用单元**
5. **重点是局部处理，不是通用修图**

## 二、允许出现的功能范围

请只围绕下面这些能力设计：

* 批量导入图片
* 导入文件夹
* 选择样图
* 手动矩形框选处理区域
* 拖拽移动、四角缩放、重新框选
* 区域列表与画布联动
* 定位方式：
  * 右下角锚定
  * 按比例定位
  * 固定像素
* 处理方式：
  * AI修复（LaMa）
  * 模糊覆盖
  * 纯色填充
  * 裁切
* 单张样图预览
* 原图 / 处理后对比滑杆
* 保存模板
* 应用已有模板
* 批量执行
* 历史任务复用
* 设置页（默认输出目录、默认格式、模型缓存等）

### 第一期约束

请在设计中默认体现这些约束：

* **默认只处理 1 个区域**
* “新增区域”可以出现，但要弱化，不要成为主操作
* 默认处理方式是 **AI修复（LaMa）**
* 默认定位方式是 **右下角锚定**
* 输出设置属于 **高级项**，默认应收起或弱化

## 五、全局布局规则

### 左侧固定导航

左侧固定导航只保留：

* 首页
* 模板中心
* 历史记录
* 设置

### 顶部栏规则

顶部栏必须 **跟随当前页面上下文变化**，不要所有页面都放同一组全局动作。

例如：

* 首页顶部只需要应用名、帮助、设置等轻量入口

* 预览页只保留“返回调整 / 重新预览 / 保存模板 / 开始批量处理”
* 历史页和模板中心不要强行挂“开始批量处理”大按钮

### 关键要求

* 不要底部 TabBar
* 必须明显是桌面端工作台布局
* 必须高保真，不要线框图
* 所有页面中文
* 不要遗漏页面，不要跳号

### P01 首页 / 启动页

页面目标：

帮助用户快速开始，不出现复杂参数。

页面元素：

* 顶部：应用名、帮助、设置
* 中央大拖拽导入区
* 按钮：导入图片
* 按钮：导入文件夹
* 按钮：应用已有模板
* 最近模板
* 最近任务

交互重点：

* 首次打开时是明显空状态
* 拖拽进入时导入区高亮
* 导入成功后自动进入模板构建页
* “应用已有模板”要足够明显，不能被埋没

要求：

* 首页简洁、直接、没有复杂参数
* 不要出现促销横幅、广告卡片、推荐功能
* 如果没有最近模板或最近任务，要展示空状态文案，而不是大块空白
* 页面文案要强调：
  * 拖入图片或文件夹开始
  * 也可以直接应用已有模板快速处理

### P02 模板构建页（最核心页面）

这是整套产品最重要的页面。

#### 页面目标

让用户在这里完成第一次模板定义。

#### 布局

采用桌面端三栏布局：

#### 左侧

* 图片列表
* 搜索框
* 当前样图高亮
* 当前区域列表
* 新增区域按钮（弱化）

#### 中间

* 大图预览区
* 手动框选区域
* 拖动移动
* 四角缩放
* 缩放控制
* 自适应窗口

#### 右侧

默认只展示核心参数：

1. 区域设置
   * x / y / w / h
   * 重新框选
   * 清除区域
2. 定位方式
   * 右下角锚定
   * 按比例定位
   * 固定像素
3. 处理方式
   * AI修复（LaMa）
   * 模糊覆盖
   * 纯色填充
   * 裁切
4. 模板名称
5. 保存模板
6. 预览效果
7. 开始批量处理

高级项：

* 输出设置允许存在，但必须折叠或弱化

#### 必须体现的状态

* 首次进入时有引导提示
* 左侧默认选中第一张样图
* 尚未框选时：
  * 预览效果 / 保存模板 / 开始批量处理禁用
* 已框选未预览时：
  * 可预览
  * 可保存模板
  * 批量处理弱化
* 预览完成后：
  * 开始批量处理高亮
* 当前模板未保存时：
  * 顶部或局部出现“有未保存更改”

#### 强制要求

* 这页必须最完整
* 一看就知道怎么操作
* 必须加入 **Step 1 / Step 2 / Step 3** 式引导
* 中间预览区必须足够大
* 框选交互要直观，像截图工具
* 不要让右侧参数面板过密
* 不要出现复杂算法参数
* 不要出现自动识别结果列表
* 不要把数值输入做成页面主角，视觉重点应是“在图上框选”

* 不要再暴露复杂编辑能力

强制要求：

* 必须有左侧样张列表
