# 图片文字标识批量清理工具 PRD（桌面端 / Tauri 版）

## 1. 产品概述

### 1.1 产品名称

中文名：图像批处理大师
英文名：Batch Image Studio

### 1.2 产品定位

一款面向创作者、设计人员、运营人员的**桌面端图片批量处理工具**。
第一版重点解决：**自动识别图片中的文字标识区域，并进行批量局部清理与修复**。

产品主要用于用户对**自有素材、已获授权素材、内部生产图片**进行标准化清理与整理，支持样张预览、批量执行、模板复用与历史任务复用。

### 1.3 一句话介绍

自动识别图片中的文字标识区域，支持批量局部清理、智能修复、样张预览与模板化处理的桌面工具。

### 1.4 平台形态

* Windows
* macOS
* Linux

### 1.5 版本定位

* 第一版：自动识别文字标识区域 + 批量局部清理
* 第二版：检测能力增强 + 多区域规则 + 更强修复效果
* 第三版：智能模板推荐 + 更复杂场景识别

## 2. 产品目标

### 2.1 核心目标

帮助用户快速完成一批图片中**文字标识区域的识别、确认、清理、导出**工作，降低重复劳动，提高批量处理效率。

### 2.2 第一版目标

第一版重点完成以下闭环：

1. 批量导入图片
2. 自动识别图片中的文字标识区域
3. 支持用户手动微调识别框
4. 对识别区域执行局部清理与修复
5. 样张预览确认效果
6. 批量执行并导出结果
7. 保存模板并复用

### 2.3 非目标

第一版暂不做：

* 在线协作
* 云端存储
* 团队权限
* 社区分享
* 复杂语义编辑
* 通用 AI 图像编辑平台

## 3. 目标用户

### 3.1 核心用户

1. 自媒体内容创作者
2. 电商运营人员
3. 设计助理
4. 素材整理人员
5. 批量处理图片的办公用户

### 3.2 用户特点

* 有明确批量处理诉求
* 需要快速处理几十到几百张图片
* 更重视效率与一致性，而不是单张精修
* 偏向桌面办公场景
* 可接受“先识别、再确认、再批处理”的流程

### 3.3 典型用户痛点

1. 同类图片中经常存在固定角落或半固定位置的文字标识
2. 一张一张手工处理效率太低
3. 通用修图软件批量能力差，难复用参数
4. 需要样张先验证，避免批量处理失败
5. 处理逻辑需要保存，下次继续用

## 4. 核心问题定义

### 4.1 当前用户问题

用户手头有大量本地图片，需要对其中的**角落文字、底边文字、固定短文字标识**做统一识别和局部清理，但目前常见工具存在这些问题：

* 自动识别能力不足
* 批量处理操作繁琐
* 很难将规则模板化
* 处理前缺少样张预览
* 失败项不好定位与重试

### 4.2 本产品解决的问题

本产品主要解决：

* 自动找出图片中的文字标识区域
* 支持固定位置、自动识别、混合模式
* 支持手动微调识别框
* 对识别区域做统一局部处理
* 一次处理多张图片
* 将识别与处理规则保存为模板

## 5. 使用场景

### 5.1 场景一：固定角落文字标识清理

用户导入一批尺寸一致的图片，文字标识大多位于右下角。
系统自动检测后，用户只需确认样张，即可批量处理。

### 5.2 场景二：半固定位置短文字清理

用户导入的图片尺寸接近，但文字位置略有偏移。
系统通过自动识别找出区域，用户微调后批量执行。

### 5.3 场景三：统一模板复用

用户已经处理过一批类似图片，希望将本次规则保存为模板，后续一键应用。

### 5.4 场景四：失败样本回查

部分图片识别失败或清理效果不好，用户需要筛选失败项重新识别、单独重试。

## 8.3 识别模块

### 功能目标

自动找出图片中的文字标识区域。

### 检测模式

1. 固定位置模式
2. 自动识别模式
3. 混合模式

### 固定位置模式

适合尺寸统一、文字位置固定的图片。
用户设定相对坐标区域后，对所有图片按同一规则处理。

### 自动识别模式

适合文字位置略有偏差的情况。
系统自动识别图片中的短文字标识区域，并生成候选识别框。

### 混合模式

先按固定区域范围限制，再在范围内自动识别，提高准确率与速度。

### 功能点

* 自动识别按钮
* 检测阈值设置
* 检测区域列表
* 当前识别区域高亮
* 识别区域数量展示
* 重新识别
* 批量应用识别规则

### 交互要求

* 识别完成后，中间预览区展示高亮框
* 右侧面板展示识别结果列表
* 选中某区域时，预览区同步高亮
* 支持对识别框拖拽、缩放、删除、新增

---

## 8.4 局部清理模块

### 功能目标

对识别出的文字区域进行统一处理。

### 清理方式

1. 智能修复
2. 模糊覆盖
3. 纯色覆盖
4. 裁切

### 说明

* 智能修复：适合尽量保留背景连续性
* 模糊覆盖：适合对细节要求不高的快速处理
* 纯色覆盖：适合背景简单区域
* 裁切：适合文字位于边角且不影响主体内容

### 功能点

* 选择清理方式
* 每个识别区域可单独设置
* 对所有区域统一设置
* 预览处理效果
* 单张图片局部重试

### 交互要求

* 样张处理中可切换不同清理方式对比
* 显示处理耗时
* 若处理失败，提示原因

# 15. 风险与注意事项

## 15.1 产品表述风险

产品文案应统一为：

* 文字标识区域识别
* 局部清理
* 智能修复
* 批量处理

避免在产品定义、页面文案、对外宣传中直接写成针对特定第三方平台的去标识工具。

## 15.2 技术风险

* 自动识别准确率不足
* 复杂背景下修复效果不稳定
* 大批量处理时性能压力大
* 不同尺寸图片的规则复用效果差异

## 15.3 体验风险

* 参数过多导致上手门槛高
* 自动识别结果不稳定时用户信任感下降
* 样张预览不充分会导致批量返工
